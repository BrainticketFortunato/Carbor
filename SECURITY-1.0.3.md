# Sicherheitsrichtlinie

Sicherheitsrelevante Probleme sind insbesondere Offenlegung von API-Schlüsseln, unbereinigte private Daten in Analyseberichten, unsichere Pfadverarbeitung und unerwartete Befehlsausführung.

Bis ein privater Meldekanal eingerichtet ist, bitte nur ein öffentliches Issue ohne vertrauliche Details anlegen und dort um einen privaten Kontaktweg bitten. Keine echten Schlüssel oder privaten Logs hochladen.
